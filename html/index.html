Catcher API 2.0

Popis a dokumentace skórovacího systému pro soutěže v Ultimate Frisbee

Autentizace a autorizace

Uživatel se přihlašuje svým emailem a heslem. Po přihlášení je mu sdělen přístupový token, kterým lze přistupovat až do vypršení jeho platnosti.

Každý zdroj a jeho požadavky mají rozdílná oprávnění. U každého požadavku je tak zobrazen seznam rolí, které mají právo požadavek volat.

Uživatelé s rolí club a organizer můžou upravovat pouze vlastní zdroje.

Formáty komunikace

Jediným podporovaným formátem je JSON. V dané hlavičce úvádějte MIME type application/json.

U typů datetime a timestamp jsou vyžadována data ve formátu YYYY-MM-DD, respektive YYYY-MM-DDTHH:MM:SS.

Hvězdička u popisů znamená povinnost být součastí požadavku.

/login

Přihlášení

POST - Přihlášení

public

Po úspěšném přihlášení jsou vráceny uživatelská data včetně přístupového tokenu pro následující požadavky. Platnost tokenu je prozatím nekonečná.

Popis polí požadavku

email* string email
password* string heslo

Popis polí úspěšné odpovědi

apiKey string přístupový token s omezenou platností (prozatím nekonečná)
email string email
password string heslo
role string role
createdAt timestamp čas vytvoření uživatele
clubId int v případě klubového účtu je vráceno ID klubu

PUT - Upraví bod

organizer admin

Slouží pro zpětnou editaci. Pro chybně zadané body je v průběhu zápasů doporučeno body smazat a znovu vytvořit. Nelze totiž upravit skóre (tj. změna týmu, který dal bod). V tom případě je nutné daný bod i všechny násedující vymazat a vytvořit je znovu.

Popis polí požadavku

order* int bod, který má být upraven
scorePlayerId int ID skórujícího hráče
assistPlayerId int ID asistujícího hráč
callahan boolean Callahan

DELETE - Smaže poslední uložený bod

organizer admin

Požadavek

Host: catcher.zlutazimnice.cz

HTTP/1.1 200 OK

POST - Udělí SOTG

club organizer admin

Hodnocení Spirit of the Game může tým (klubový účet) udělit jinému týmu pouze za předpokladu, že s ním odehrál zápas. Hodnocení všem zápasům může zadávat i organizátor.

Popis polí požadavku

matchId int ID týmu, jenž má být ohodnocen
comment string hodnocení SOTG: komentář
rules int hodnocení SOTG: znalost pravidel (0 až 4)
fouls int hodnocení SOTG: fauly a tělesný kontakt (0 až 4)
fair int hodnocení SOTG: férové smýšlení (0 až 4)
positive int hodnocení SOTG: pozitivní přístup (0 až 4)
communication int hodnocení SOTG: komunikace (0 až 4)
