Formáty komunikace

/club/{id}

/tournament/{id}

Turnaj

GET - Informace o konkrétním turnaji

public organizer admin

Získá základní informace o turnaji.

Parametry
standings boolean K výsledku přiřadí výsledné pořadí týmů.

PUT - Úprava turnaje

organizer admin

Nejdůležitejší úpravou je změna příznaku active na true. Tím dojde k přiřazení týmů na jejich místa podle nasazení a turnaj se považuje za připravený.

/tournament/{id}/players

Hráči na turnaji

GET - Seznam hráčů

public team organizer admin

Vrátí seznam hráčů, včetně jejich statistik. Volitelným parametrem je teamId, který zmenší výslednou množinu na hráče jednoho týmu.

POST - Přidání hráče k týmu

team organizer admin

Vytváří vazbu mezi týmem a hráčem. Lze tak individuálně přídávat hráče na soupisku.

Požadavek
POST /api/tournament/1/players HTTP/1.1
Host: catcher.zlutazimnice.cz
Content-Type: application/json

{
  "playerId": 999,
  "teamId": 1
}
Odpověď
HTTP/1.1 201 Created
{
  UNFINISHED
}

DELETE - Smazání hráče z týmu

team organizer admin

Lze smazat pouze takového hráče, jenž nemá žádný odehraný zápas.

Požadavek
DELETE /api/tournament/1/players HTTP/1.1
Host: catcher.zlutazimnice.cz
Content-Type: application/json

{
  "playerId": 999,
  "teamId": 1
}
Odpověď
HTTP/1.1 200 OK

/tournament/{id}/teams

Týmy na turnaji

GET - Seznam týmů

public team organizer admin

Vrátí seznam týmů, včetně jejich statistik.

Parametry
teamId int ID týmu, jenž zmenší výslednou množinu dat na data jednoho týmu.
players boolean Příznak, zda k odpovědi přiřadit seznam hráčů včetně jejich statistik.

PUT - Změna účastníka turnaje

team organizer admin

Jestliže turnaj nemá příznak active, lze stále přidávat nebo měnit kteréhokoliv účastníka turnaje.

Tělo požadavku
seeding int Pozice, na kterou bude nový tým dosazen.
teamId int ID nového týmu.
Požadavek
   PUT /api/tournament/1/teams HTTP/1.1
   Host: catcher.zlutazimnice.cz
   Content-Type: application/json
   
   {
     "seeding": 2,   
     "teamId": 3
   }
Odpověď
HTTP/1.1 200 OK

/tournament/{id}/matches

Zápasy na turnaji

GET - Seznam zápasů

public team organizer admin

Vrátí seznam zápasů.

Parametry
matchId int ID zápasu, jenž zmenší výslednou množinu dat na data jednoho konkrétního zápasu.
fieldId int ID hřiště, jenž zmenší výslednou množinu dat na seznam zápasů z jednoho konkrétního hřiště.
points boolean Příznak, zda ke každému zápasu přiřadit seznam všech odehraných bodů.
date datetime Zápasys konkrétního dne (ve formátu YYYY-MM-DD).
active boolean Příznak, zda vrátit pouze právě odehrávaná utkání.
terminated boolean Příznak, zda vrátit pouze již odehraná utkání.

PUT - Změna zápasu

team organizer admin

Nejčastěji půjde o ukončení zápasu parametrem terminate. Dále pak parametry terminate

/tournament/{id}/spirit

Spirit of the Game
Bude vracet a ukladat spirit per match, per team.

/tournament/{id}/points

Body na turnaji
Bude vracet a ukladat body per match.

/tournaments

Turnaje

GET - Seznam všech turnajů

public organizer admin

Parametry
from datetime Turnaj se odehrává po tomto datumu (ve formátu YYYY-MM-DD).
to datetime Turnaj se odehrává před tímto datumem (ve formátu YYYY-MM-DD).
country string Země, ve které se turnaj odehrává (ve formátu ISO 3166-1 alpha-3).
divisionId int Identifikátor divize.
active boolean Příznak turnaje, zda je aktivní.
terminated boolean Příznak turnaje, zda je ukončený.
cald boolean Příznak, zda jde o turnaj ČALD.
Požadavek
GET /api/tournaments HTTP/1.1
Host: catcher.zlutazimnice.cz
Odpověď
HTTP/1.1 200 OK
{
  "count": 2,
  "items": [
    {
      UNFINISHED
    },
    ...
  ]
}

POST - Vytvoření nového turnaje

organizer admin

Požadavek je nejprve důkladně zkontrolován, zda je hratelný a neobsahuje chyby, a pak je uložen do databáze. Po vytvoření je nutné turnaj aktivovat, tzn. nastavit příznak active na hodnotu true.

Tělo požadavku
startDate datetime První hrací den turnaje (ve formátu YYYY-MM-DD).
endDate datetime Poslední hrací den turnaje (ve formátu YYYY-MM-DD).
Požadavek
POST /api/tournaments/ HTTP/1.1
Host: catcher.zlutazimnice.cz
Content-Type: application/json

{
  UNFINISHED
}
Odpověď
HTTP/1.1 201 Created
{
  UNFINISHED
}
