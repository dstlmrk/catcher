Autentizace a autorizace

Uživatel se přihlašuje ...

Pod titulkem každého požadavku je seznam všech rolí, které mají právo požadavek zavolat. Ne všechny role mají však u nekterých požadavků všechny jeho možnosti. U role s omezením je použito modré odlišení.

Formáty komunikace

Jediným podporovaným formátem je JSON. V dané hlavičce úvádějte MIME type application/json.

U typů datetime a timestamp jsou vyžadována data ve formátu YYYY-MM-DD, respektive YYYY-MM-DDTHH:MM:SS.

/tournament/{id}

Turnaj

GET - Informace o konkrétním turnaji

public club organizer admin

Získá základní informace o turnaji.

PUT - Úprava turnaje

organizer admin

Nejdůležitejší úpravou je změna příznaku ready, active nebo terminated (UNFINISHED) na true. První změnou dojde k přiřazení týmů na jejich místa podle nasazení a turnaj se považuje za připravený. Druhý příznak označí turnaj jako právě probíhající a třetí přiznak turnaj ukončí a zveřejní hodnocení SOTG všem uživatelům.

Dále jsou editovatelné atributy name, startDate, endDate, city, country, caldTournametId.

UNFINISHED /tournament/{id}/spirit

Spirit of the Game

GET - Získá hodnocení SOTG

public team organizer admin

Díky dvěma parametrům lze získat kterékoliv hodnocení Spirit of the Game. Turnaj však musí mít příznak terminated. Bez něj jsou výsledky veřejné pouze pro roli organizer. Ke svým odevzdaným hodnocením má vždy přístup role team.

Parametry
matchId int Zmenší výslednou množinu na údaje jednoho zápasu.
teamId int Zmenší výslednou množinu na údaje jednoho zápasu.

POST - Udělí SOTG

team organizer admin

Hodnocení Spirit of the Game může team udělit jinému týmu pouze za předpokladu, že s ním odehrál zápas. Hodnocení všem zápasům může zadávat i organizer.

Tělo požadavku
matchId int ID týmu, jenž má být ohodnocen.
teamId int ID zápasu, kterého se hodnocení týká.
comment string Hodnocení SOTG: komentář.
rules int Hodnocení SOTG: znalost pravidel.
fouls int Hodnocení SOTG: fauly a tělesný kontakt.
fair int Hodnocení SOTG: férové smýšlení.
positive int Hodnocení SOTG: pozitivní přístup.
communication int Hodnocení SOTG: komunikace.
Požadavek
POST /api/tournament/1/teams HTTP/1.1
Host: catcher.zlutazimnice.cz
Content-Type: application/json

{
  "matchId": 2,   
  "teamId": 3,
  "comment": "Game was very agresive, but fair",
  "rules": 3
  "fouls": 1
  "fair": 4
  "positive": 2
  "communication": 2
}
Odpověď
HTTP/1.1 201 Created
{
  UNFINISHED
}

POST - Započítá nový bod

organizer admin

Tělo požadavku
homePoint boolean povinný příznak, zda domácí tým dal bod
scorePlayerId int nepovinný ID skórujícího hráče
assistPlayerId int nepovinný ID asistujícího hráč
callahan boolean nepovinný Callahan
Požadavek
POST /api/tournament/1/match/1/points HTTP/1.1
Host: catcher.zlutazimnice.cz
Content-Type: application/json

{
  "assistPlayerId": 3,   
  "scorePlayerId": 2,
  "homePoint": true,
  "callahan": false
}
Odpověď
HTTP/1.1 201 Created
{
  "awayScore": 3,

  "homePoint": true,
  "scorePlayerId": 2,
  "homeScore": 7,
  "callahan": false,
  "assistPlayerId": 3,
  "order": 9
}

PUT - Upraví bod

organizer admin

Slouží pro zpětnou editaci. Pro chybně zadané body je v průběhu zápasů doporučeno body smazat a znovu vytvořit. Nelze totiž upravit skóre (tj. změna týmu, který dal bod). V tom případě je nutné daný bod i všechny násedující vymazat a vytvořit je znovu.

Tělo požadavku
order int povinný bod, který má být upraven
scorePlayerId int nepovinný ID skórujícího hráče
assistPlayerId int nepovinný ID asistujícího hráč
callahan boolean nepovinný Callahan

DELETE - Smaže poslední uložený bod

organizer admin

Smaže poslední uložený bod. Opakovaným voláním lze smazat všechny již dříve uložené body. Tento požadavek lze volat pouze na zápasy, které ještě nebyly ukončeny.

Požadavek
DELETE /api/tournament/1/match/12/points HTTP/1.1
Host: catcher.zlutazimnice.cz
Odpověď
HTTP/1.1 200 OK
